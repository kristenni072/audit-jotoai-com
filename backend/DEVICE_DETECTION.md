# 客户端类型检测功能

## 功能说明

后端会自动检测用户提交联系表单时使用的设备类型，并将信息保存到留言记录和邮件通知中。

## 检测逻辑

基于HTTP请求头中的 `User-Agent` 字段自动识别：

- **手机**: iPhone, Android, Windows Phone等移动设备
- **平板**: iPad, Android Tablet等平板设备  
- **电脑**: 桌面浏览器（默认）
- **未知**: 无法识别User-Agent时

## 数据存储

每条联系表单记录会包含 `deviceType` 字段：

```json
{
  "name": "张三",
  "email": "test@example.com",
  "phone": "13800138000",
  "source": "直接访问",
  "deviceType": "手机",  // 新增字段
  "submittedAt": "2026-02-26T08:00:00.000Z"
}
```

## 邮件通知

邮件通知中会显示客户端类型信息：

```
新的联系表单提交

姓名：张三
学校/机构：测试学校
邮箱：test@example.com
电话：13800138000
留言：测试留言

---
访问来源：直接访问
客户端类型：手机  ← 新增
提交时间：2026-02-26 16:00:00
```

## 飞书表格同步

如果配置了飞书表格同步，客户端类型信息会自动同步到飞书表格中，方便进行数据统计和分析。

飞书表格中会新增一列：
- **列名**: `客户端类型` 或 `deviceType`
- **值**: 手机 / 平板 / 电脑 / 未知

## 前端无需修改

此功能完全在后端实现，前端无需做任何修改。后端会自动从HTTP请求头中提取设备信息。

## 统计分析

管理员可以在后台查看留言列表时，看到每条留言的设备类型，用于分析用户行为：

- 了解用户主要使用什么设备访问网站
- 针对不同设备优化用户体验
- 分析不同设备的转化率
